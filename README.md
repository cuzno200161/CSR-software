# Overview

Most scientific and engineering challenges, including personalized medicine, fall within the domain of complex systems, an emergent field of physics that gained prominence in the 1970s. Complex systems consist of numerous interacting components that self-organize into dynamic, nonlinear structures exhibiting emergent properties.

Directly linking these emergent properties to the underlying interacting components remains extremely challenging due to the intrinsic complexity of the self-organization process. Reductionist reasoning has long been the prevailing paradigm in both fundamental and applied sciences, including the study of complex systems. This approach begins from first principles and relies on mechanistic models to simulate self-organization. However, such models typically capture only partial aspects of the system’s behavior. Even when they successfully reproduce emergent properties, their applicability is inherently constrained by the assumptions embedded in the underlying mechanisms.

### The Inductive Approach
Inspired by Isaac Newton’s inductive reasoning prior to formulating the laws of motion, we adopt a similar strategy to infer emergent properties from patterns observed in nature. Specifically, the relationship between emergent properties (e.g., therapeutic efficacy in medicine) and parameter space (e.g., drug-dose combinations) exhibits a smooth, continuous surface that can be represented by the Complex Systems Response (CSR) equation.

Optimization in high-dimensional parameter spaces is challenging because the number of possible parameter combinations grows exponentially. For a system with $P$ parameters and $M$ levels for each parameter, the traditional search space forms a huge $M^P$ space. If a trial-and-error approach is adapted, it is a daunting task to search through the entire space for the optimal parameter-level combination.

Through artificial neural networks analysis of a set of test data, the input-output response was found to be a smooth surface. The Complex Systems Response (CSR) Equation provides a deterministic, second-order nonlinear framework for modeling and optimizing complex systems using only a limited number of calibration experiments. Using Orthogonal Array Composite Design (OACD), the required number of experiments is dramatically reduced to:

$$\frac{P^2 + 3P + 2}{2}$$

instead of exhaustively searching the entire parameter space.

# Terminology

## 1. Complex Systems Response (CSR) Equation
A deterministic, second-order nonlinear equation that quantitatively maps how multiple interacting parameters lead to an emergent system property. It acts as the transfer function between system inputs and outputs.

## 2. CSR Response Surface
A multidimensional surface defined by the CSR equation that visualizes how the emergent property changes with varying parameters. It can take the form of an elliptic or hyperbolic paraboloid, typically featuring a single global optimum.

## 3. Parameters, Parameter Levels, and Parameter Range
* **Parameters:** Controllable factors in the complex system that are of interest to researchers.
* **Parameter Levels:** Specific values of the parameters that researchers choose when investigating a complex system.
* **Parameter Range:** Defined strictly by the minimum and maximum of the parameter levels.

## 4. CSR Coefficients ($x_0, x_i, x_{ii}, x_{ij}$)
Adaptive parameters in the CSR equation that represent system response characteristics. Unlike fixed regression coefficients, CSR coefficients depend on time and hidden variables, making them dynamic descriptors of system behavior:
* `x₀`: Baseline response or intercept term, representing the system’s intrinsic state when all controllable parameters are zero.
* `x\ᵢ`: First-order response coefficient showing the direct influence of parameter $c_i$ on the emergent property.
* `x\ᵢ\ᵢ`: Second-order self-interaction term that captures nonlinearity of parameter $c_i$’s individual effect.
* `x\ᵢ\ⱼ`: Cross-interaction coefficient quantifying the coupling effect between parameters $c_i$ and $c_j$.

## 5. Orthogonal Array Composite Design (OACD)
A design-of-experiments (DOE) method that determines the minimal number of calibration tests needed to estimate CSR coefficients. For $P$ parameters, only $(P^2 + 3P + 2)/2$ tests are required instead of exploring all possible combinations.

## 6. Calibration (Analysis) Process
The procedure of determining CSR coefficients by analyzing experimental data with the CSR equation.

## 7. Multiple Objective Optimization (MOO)
When there are more than one outcomes that need to be optimized, multiple objective optimization is applied. This optimization process makes sure that each outcome to be optimized is considered and gives a balanced result where each outcome is as optimized as possible.

## 8. Extremum (Global Optimum)
The maximum or minimum point on the CSR surface representing the best achievable system performance under given constraints.

## 9. Normalization
Normalization reduces the parameter range to standard scale intervals of either `[-1, 1]` or `[0, 1]`.

# CSR Equation Framework

The mathematical definition of the Complex Systems Response function is expressed as a second-order nonlinear response equation:

$$E(c_i, u_x, t) = x_0(u_x, t) + \sum_{i=1}^{P} x_i(u_x, t)c_i(t) + \sum_{i=1}^{P} x_{ii}(u_x, t)c_i^2(t) + \sum_{1 \le i < j \le P} x_{ij}(u_x, t)c_i(t)c_j(t)$$

Where:
* $c_i$ represents the explicit level of a parameter and is an independent variable.
* $E(c_i, u_x, t)$ is the observed outcome.
* $x_i, x_{ii}, x_{ij}$ are the unknown response coefficients determined through experimental data analysis. These coefficients contain the information of the system response to the parameters and the levels, are functions of the independent variables (i.e. time and $c_i$), and hence are not constants.
* $u_x$ represents undefined or unknown background parameters.
* $t$ represents time.

By definition, this is a mechanism-free formulation. For $P$ parameters, the exact number of coefficients to solve is:

$$1 + P + P + \frac{P(P - 1)}{2} = \frac{P^2 + 3P + 2}{2}$$

This saves orders of magnitude of effort, time, and cost compared to searching through the entire search space.

# Workflow

## 1. OACD Table Generation
Use OACD generation to create correct runs for your experiment.

### Steps:
1. Navigate to the **OACD Table Builder** tab.
2. Select **Number of Factors** – number of variables in the system.
3. Select **Table Size** – this scales the number of runs in the experiment.
4. Under **Factor Min/Max**, set min and max values for each factor. Alternatively, you can import extremum from a `.xlsx` file.
5. If you only need a certain amount of factors to be non-zero, you can remove all experiments with more using **Max Nonzero Factors**.
6. If a limitation is needed (ex: `C1 + C2 + C3 < 100`), add a limit in **Parameter Limits**, then select that limit next to the corresponding factors in Factor Min/Max.
7. Click **Generate OACD Table!** You may then **Export as Excel** to have your experiment runs as a `.xlsx` file.

---

## 2. Load Experimental Data
1. Navigate to the **CSR Integration** tab.
2. Click on **Select Data File**.
3. Choose the `.xlsx` file containing experimental data.

### Input Requirements:
* The `.xlsx` file must specify the names of all factors and results in the first row.
* Subsequent rows contain the corresponding numerical data entries.

#### Spreadsheet Input Example:
| Temperature T (°C) | Energy density E ($\text{J/mm}^3$) | Compressive Yield Stress CYS (MPa) |
| :--- | :---: | :---: |
| 160 | 52.8 | 437.1 |
| 160 | 111.1 | 534.2 |
| 25 | 52.8 | 413.1 |
| 25 | 111.1 | 534.1 |

---

## 3. Configure Settings
1. **Global Optimum:** Select between `Maximum`, `Minimum`, `Maximum absolute value` and `Minimum absolute value`.
2. **Normalization:** Select between `[-1, 1]` and `[0, 1]`.
3. **Parameter or Outcome?:** Under *Parameter or Outcome?*, select if the column belongs to parameter, outcome, or ignore. If more than one outcome columns are chosen, then multiple objective optimization is automatically activated.
4. **Parameter Limits:** If a parameter limitation is to be set (for example, `C1 + C2 + C3 < 100`), navigate to the *Parameter Limits Section*, click the checkbox if the factor should be included in that limit, set the limit value at *CSR Factor Limits*, and click *add limit*. If a limit needs to be removed, click on *Remove selected* or *Clear all*.
5. After all is set, click on **Run Analysis Process**.

---

## 4. Output and Analysis
After the process completes, the software populates the interface with the following analytical modules:

* **CSR Equation:** Displays the numerical expression of the analyzed equation.
* **Parameter Definitions:** Lists the names of the parameters used in the function.
* **Extremum:** Indicates the maximum/minimum point of the plot. If multiple result columns are selected, then *Individual Results at Extremum* will be shown below.
* **Model Analysis (Statistics):** Provides the R-square and root mean square error (RMSE) value of the model.
* **Actual vs. Predicted Values:** Presents the deviations between predicted outcomes and observed data.
* **CSR Response Surface Plot:** Provides a graphical representation of the analyzed CSR function.
* **Coefficient Analysis:** Navigate to the *Coefficient analysis* tab after the analysis run is complete. In *Analysis Controls*, select whether the coefficient shall be determined when the factors are at minimum, maximum, or extremum. The pie charts demonstrate the distribution of the coefficient absolute values in terms of linear ($x_i$), quadratic ($x_{ii}$), and interaction ($x_{ij}$) terms.

# Applications

The CSR framework operates on a mechanism-free formulation, allowing direct deployment across diverse domains without requiring detailed prior knowledge of underlying mechanistic pathways:

* **Biomedical Optimization:** First identified in a cell-line study of non–small cell lung cancer treated with combinatorial drug regimens, it has since been generalized and validated across approximately 30 disease models, including cellular systems, animal models, and clinical investigations. It is particularly powerful for biomarker discovery and combination therapy optimization, where system complexity and patient variability often limit mechanistic modeling. Please refer to the paper below for examples demonstrating how CSR can identify clinically relevant biomarker sets:
  * X.T Ding, P. Krutzik, A. A. Ghaffari, Y. Zhaozhi, D. Miranda Jr., G. Cheng, C. -M. Ho, G. P. Nolan, D. J. Sanchez, *“Cellular Signaling Analysis shows antiviral, ribavirin-mediated ribosomal signaling modulation”*, **Antiviral Research** 171:104598, (2019), doi: 10.1016/j.antiviral.2019.104598.
* **Engineering & R&D Workflows:** Enables seamless integration into existing R&D and industrial workflows, facilitating rapid deployment for drug development, engineering products, materials design, chemical systems, and manufacturing process optimization.
* **Socio-Physical Systems:** Extends as a unified framework to physical, chemical, social, and industrial complex systems, demonstrating its broad applicability.
