# Overview

Optimization in high-dimensional parameter spaces is challenging because the number of possible parameter combinations grows exponentially.

For a system with:

* `P` parameters
* `M` levels for each parameter

The search space becomes:

[
M^P
]

Traditional trial-and-error methods quickly become impractical.

The **Complex Systems Response (CSR) Equation** provides a deterministic, second-order nonlinear framework for modeling and optimizing complex systems using only a limited number of calibration experiments.

Using Orthogonal Array Composite Design (OACD), the required number of experiments is reduced to:

[
\frac{P^2 + 3P + 2}{2}
]

instead of exhaustively searching the entire parameter space.

# Terminology

## 1. CSR Equation (Complex Systems Response Equation)

A deterministic, second-order nonlinear equation that quantitatively maps how multiple interacting parameters produce an emergent system property.

The CSR equation acts as the transfer function between system inputs and outputs.

---

## 2. CSR Response Surface

A multidimensional response surface defined by the CSR equation.

It visualizes how the emergent property changes with varying parameters and may take forms such as:

* Elliptic paraboloid
* Hyperbolic paraboloid

Typically, the surface contains a single global optimum.

---

## 3. Parameters, Parameter Levels, and Parameter Range

### Parameters

Controllable variables in the system.

### Parameter Levels

Specific values selected for each parameter.

### Parameter Range

The minimum and maximum values allowed for each parameter.

---

## 4. CSR Coefficients

Adaptive coefficients representing the system response characteristics.

Unlike fixed regression coefficients, CSR coefficients depend on time and hidden variables.

### Coefficient Types

| Coefficient | Meaning                                 |
| ----------- | --------------------------------------- |
| `x₀`        | Baseline response/intercept             |
| `xᵢ`        | First-order parameter effect            |
| `xᵢᵢ`       | Second-order nonlinear self-interaction |
| `xᵢⱼ`       | Cross-interaction between parameters    |

---

## 5. Orthogonal Array Composite Design (OACD)

A design-of-experiments (DOE) method that minimizes the number of calibration tests required.

For `P` parameters:

[
\frac{P^2 + 3P + 2}{2}
]

experiments are required instead of testing all parameter combinations.

---

## 6. Calibration Process

The process of determining CSR coefficients by analyzing experimental data with the CSR equation.

---

## 7. Multiple Objective Optimization (MOO)

When multiple outcomes must be optimized simultaneously, MOO balances competing objectives to achieve the best overall solution.

---

## 8. Extremum (Global Optimum)

The maximum or minimum point on the CSR response surface.

---

## 9. Normalization

Parameter ranges can be normalized to:

* `[-1, 1]`
* `[0, 1]`

---

# CSR Equation

The CSR surface can be represented using a second-order nonlinear response equation.

For `P` parameters, the number of coefficients is:

[
1 + P + P + \frac{P(P-1)}{2}
]

which simplifies to:

[
\frac{P^2 + 3P + 2}{2}
]

This dramatically reduces experimental effort compared to brute-force optimization.

# Workflow

## 1. OACD Table Generation

Generate the experimental design table.

### Steps

1. Navigate to the **OACD Table Builder** tab
2. Select:

   * Number of Factors
   * Table Size
3. Define:

   * Minimum parameter values
   * Maximum parameter values
4. Optionally:

   * Import extrema from `.xlsx`
   * Limit the number of nonzero factors
   * Add parameter constraints
5. Click **Generate OACD Table**
6. Export the experiment table as Excel

### Example Constraint

```text
C1 + C2 + C3 < 100
```

---

## 2. Load Experimental Data

Navigate to:

```text
CSR Integration → Select Data File
```

### Input Requirements

The `.xlsx` file must:

* Use the first row for parameter and outcome names
* Use subsequent rows for experimental data

---

## 3. Configure Settings

### Global Optimum Modes

Choose one:

* Maximum
* Minimum
* Maximum absolute value
* Minimum absolute value

### Normalization Modes

Choose one:

* `[-1, 1]`
* `[0, 1]`

### Parameter Roles

Each variable can be set as:

* Parameter
* Outcome
* Ignore

If multiple outcomes are selected, Multiple Objective Optimization is automatically enabled.

### Parameter Limits

Constraints can be added during optimization.

Example:

```text
C1 + C2 < 50
```

After configuration:

```text
Click → Run Analyzing Process
```

---

## 4. Output and Analysis

After optimization completes, the software generates:

### CSR Equation

The numerical expression of the analysis model.

### Extremum

The optimal parameter combination.

### Model Analysis Statistics

* R-square
* RMSE

### Actual vs. Predicted Values

Comparison between measured and modeled outputs.

### CSR Response Surface Plot

Visualization of the analyzed response surface.

### Coefficient Analysis

Pie-chart visualization of:

* Linear terms (`xᵢ`)
* Quadratic terms (`xᵢᵢ`)
* Interaction terms (`xᵢⱼ`)
